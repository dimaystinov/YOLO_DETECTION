# YOLO_DETECTION

Детекция объектов на изображениях и видео с помощью YOLO, визуализация через OpenCV. Кадры приводятся к ширине 640 с сохранением пропорций.

Готовый проект с exe https://github.com/dimaystinov/YOLO_DETECTION/releases/download/yolo/YOLO_Detection.zip 
---

## Что используется в проекте

| Технология | Назначение |
|------------|------------|
| **Python 3** | Язык и среда выполнения. |
| **Ultralytics (YOLO)** | Загрузка обученной модели и инференс: поиск объектов на кадре, координаты боксов, классы и уверенность. |
| **OpenCV (cv2)** | Чтение изображений/видео, ресайз с сохранением пропорций, отрисовка боксов и подписей, показ результата в окне. |
| **yolo26n.pt** | Файл весов модели YOLO (nano), лежит в корне проекта. |

Итог: YOLO считает детекции, OpenCV готовит кадр и рисует результат.

## Установка

1. Клонируй репозиторий и перейди в папку проекта:
   ```bash
   cd YOLO_DETECTION
   ```

2. Создай виртуальное окружение (рекомендуется):
   ```bash
   python -m venv venv
   ```
   - Windows: `venv\Scripts\activate`
   - Linux/macOS: `source venv/bin/activate`

3. Установи зависимости из `requirements.txt`:
   ```bash
   pip install -r requirements.txt
   ```

4. Положи веса модели `yolo26n.pt` в корень проекта (если ещё нет).

---

## Запуск

```bash
python main.py
```

Скрипт читает путь к файлу из `conf.json`, запускает детекцию по изображению или видео и показывает результат в окне. Рядом с программой должны лежать `yolo26n.pt` и `conf.json`.

---

## Сборка exe (PyInstaller)

1. Установи зависимости проекта и PyInstaller:
   ```bash
   pip install -r requirements.txt
   pip install pyinstaller
   ```

2. Собери один файл exe:
   ```bash
   pyinstaller --clean YOLO_Detection.spec
   ```
   Или запусти скрипт:
   ```bash
   build.bat
   ```

3. Готовый `YOLO_Detection.exe` будет в папке `dist\`. Рядом с exe положи `yolo26n.pt` и `conf.json` (путь в конфиге — к картинке или видео).

## Подробное описание кода (`main.py`)

### Импорты и константа

- **`import cv2`** — OpenCV: работа с изображениями, ресайз, рисование, окно.
- **`from ultralytics import YOLO`** — API для загрузки модели и вызова детекции.
- **`TARGET_WIDTH = 640`** — целевая ширина кадра при ресайзе; высота считается по пропорциям.

---

### Функция `resize_keep_aspect(frame, width=TARGET_WIDTH)`

Приводит кадр к заданной ширине, сохраняя соотношение сторон.

- **Вход:** `frame` — кадр (numpy-массив BGR), `width` — нужная ширина (по умолчанию 640).
- Берёт высоту и ширину из `frame.shape[:2]`.
- Если текущая ширина уже равна `width`, возвращает кадр без изменений.
- Иначе считает новую высоту: `new_h = int(h * width / w)`.
- Делает `cv2.resize(frame, (width, new_h))` и возвращает результат.

Используется и для картинки, и для каждого кадра видео, чтобы и детекция, и отображение шли в одном масштабе.

---

### Функция `draw_detections(frame, results, model, conf_threshold=0.25)`

Рисует на кадре боксы и подписи по результатам YOLO.

- **Вход:** `frame` — кадр, на котором рисуем; `results` — результат вызова `model(...)`; `model` — экземпляр YOLO (нужен `model.names`); `conf_threshold` — минимальная уверенность для отображения (по умолчанию 0.25).
- **`results[0].boxes`** — все детекции для первого (и единственного) изображения в батче.
- **`model.names`** — словарь «id класса → строка с именем класса».

Для каждого бокса:

1. **`box.conf[0]`** — уверенность (0..1). Если меньше `conf_threshold`, бокс пропускается.
2. **`box.cls[0]`** — id класса; **`box.xyxy[0]`** — координаты в формате (x1, y1, x2, y2).
3. Собирается подпись: имя класса + уверенность, например `"person 0.87"`.
4. **`cv2.rectangle(frame, (x1, y1), (x2, y2), color, thickness)`** — рисуется прямоугольник вокруг объекта (цвет зелёный BGR (0, 255, 0), толщина 2).
5. **`cv2.getTextSize(...)`** — размер текста подписи; затем рисуется заливка под подпись (прямоугольник над боксом) и **`cv2.putText(...)`** — сама подпись чёрным текстом.

Функция меняет `frame` «на месте» и возвращает его же (удобно для цепочки вызовов).

---

### Функция `main()`

Точка входа: загрузка модели, один из двух режимов (картинка или видео), показ результата.

1. **Загрузка модели:** `model = YOLO("yolo26n.pt")` — веса из файла в корне проекта.

2. **Режим «одна картинка» (Вариант 1):**
   - Путь к изображению: `src/img.jpg`.
   - **`cv2.imread(image_path)`** — загрузка; при ошибке выводится сообщение и выход.
   - **`resize_keep_aspect(frame)`** — ресайз до ширины 640 с сохранением пропорций.
   - **`model(frame, verbose=False)`** — детекция; вывод в консоль отключён.
   - **`draw_detections(frame, results, model)`** — отрисовка боксов и подписей на том же кадре.
   - **`cv2.imshow("YOLO + OpenCV", frame)`** — показ окна с результатом.
   - **`cv2.waitKey(0)`** — ожидание любой клавиши.
   - **`cv2.destroyAllWindows()`** — закрытие окна.

3. **Режим видео (Вариант 2, закомментирован):**
   - **`cv2.VideoCapture(r"src\apples.mp4")`** — открытие видеофайла.
   - В цикле: **`cap.read()`** даёт следующий кадр; при конце видео выход из цикла.
   - Для каждого кадра: ресайз через `resize_keep_aspect(frame)`, вызов `model(...)`, `draw_detections(...)`, показ в том же окне.
   - **`cv2.waitKey(1)`** — короткая пауза между кадрами; при нажатии **q** выход по `ord("q")`.
   - В конце: **`cap.release()`** и **`cv2.destroyAllWindows()`**.
